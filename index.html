My first GitHub Pages site

03 Jan 2026 · 4 min read · Shipping

I wanted a simple option to document my tech journey - honing my SQL and Python DS/ML skills, and learning Node.js & React. GitHub Pages was the best option because it didn't require setup on another platform and made it easy to share my project updates.

The context

I've been learning across a few areas at once and I needed somewhere to capture progress properly. Not polished tutorials. Not a stream of half-thoughts. Just short write-ups that I can come back to later.

The problem

The confusing bit wasn't "how to make a website". It was GitHub itself. Repositories, commits, branches, and why a file name like index.html matters. I could feel I was one small detail away from it working, but the details were exactly what I didn't know yet.

The investigation

  • I checked what GitHub Pages actually serves by default.
  • I looked at where the Pages settings point to: branch and folder.
  • I verified the files existed in the right place and were named correctly.
  • I refreshed a few times and learnt the hard truth about browser caching.

The fix

The fix was simple once I stopped guessing and followed the flow:

  1. Create index.html in the repo root.
  2. Enable Pages in Settings → Pages and deploy from the main branch root.
  3. Add a stylesheet and link it properly.

The key line that made styling work was:

<link rel="stylesheet" href="styles.css" />

What I learned

  • GitHub Pages is predictable. It only feels mysterious when you don't know what it expects.
  • File names and locations matter. A missing index.html is enough to make everything look broken.
  • Commits are basically "save points" for your site. Commit equals deploy.
  • If a change doesn't show, it's usually caching or the Pages build still catching up.

Next time

  • I'll add pages deliberately instead of linking to pages that don't exist.
  • I'll keep the structure consistent so writing stays easy.
  • I'll treat the site as a system, not a one-off task.

Next step

I'm going to publish my next post using the same structure, then improve the Posts page so it lists posts cleanly as the site grows.
